# 手势识别与硬件交互系统技术解析报告

## 一、 项目背景与目标
本项目旨在开发一个能够实时识别复杂手势并控制外部硬件的智能交互系统。通过摄像头捕获手部动态，系统能够精准定位 21 个手部关键点，并利用机器学习模型判断手势含义，同时输出平滑后的坐标指令，用于驱动云台、机械臂等硬件设备。

## 二、 系统架构与工作流程
项目遵循"数据驱动"的设计思路，主要分为三个核心阶段：

1.  **数据采集阶段 (`collect_data.py`)**：
    - 利用 `HandTrackingModule.py` 提取手部关键点坐标。
    - 记录 21 个地标点的 $(x, y)$ 坐标，并将其展平为 42 维特征向量。
    - 将特征向量与用户自定义的标签（如 "OK", "Like"）关联，存入 `dataset.csv`。

2.  **模型训练阶段 (`train_model.py`)**：
    - 加载 CSV 数据集，使用 **Scikit-learn** 进行数据预处理（标准化）。
    - 训练 **SVM（支持向量机）** 分类器，并将训练好的模型导出为 `gesture_model.pkl`。

3.  **实时推理与控制阶段 (`inference.py` / `HandTrackingModule.py`)**：
    - 实时从摄像头读取视频流。
    - 加载模型，对当前帧的手势进行毫秒级分类预测。
    - 提取特定点（如大拇指尖）坐标，经过平滑滤波后通过串口（Serial）发送给硬件。

## 三、 核心算法深度解析

### 1. 手部关键点追踪 (MediaPipe Hands)
项目采用了 Google 的 **MediaPipe Hands** 解决方案：
- **BlazePalm 检测器**：首先在全图中定位手掌区域。
- **Hand Landmark 模型**：在手掌区域内回归出 21 个 3D 坐标点。
- **优势**：该算法对不同光照、背景复杂度以及手部旋转具有极强的鲁棒性，是目前移动端和桌面端最先进的实时追踪方案。

### 2. 手势分类算法 (SVM - 支持向量机)
系统核心分类器采用了 **支持向量机 (SVC)**：
- **特征工程**：输入特征为 21 个点的像素坐标（经过标准化）。
- **核函数 (Kernel)**：使用线性核函数（Linear Kernel）。SVM 通过寻找特征空间中的最优超平面，使得不同手势类别之间的分类间隔（Margin）最大化。
- **选择理由**：相较于深度神经网络，SVM 在特征维度较低（42维）且样本量较小时，具有更快的推理速度（实时性极高）且不易过拟合。

### 3. 信号平滑与抖动抑制
在硬件控制中，摄像头捕捉的原始坐标通常存在高频抖动。项目实现了两种滤波策略：
- **滑动平均滤波 (Moving Average Filter)**：维护一个长度为 5 的历史窗口，取均值输出，使坐标移动更加丝滑。
- **离群点剔除 (Outlier Detection)**：计算当前坐标与历史均值的偏差。若 `abs(diff) > 50`，则判定为瞬间误检，强制沿用历史值，避免硬件产生剧烈撞击或震动。

### 4. 启发式逻辑分类 (`FingerCounter.py`)
除了机器学习，项目还包含了一种基于几何关系的计数算法：
- **逻辑核心**：通过比较指尖（Tip ID）与其下方第二个关节（Pip ID）的 $y$ 坐标相对位置，判断手指是否处于伸展状态。
- **特点**：无需训练，响应速度极快，适用于简单的数字识别。

## 四、 实现细节与通信协议
- **数据格式**：系统在训练和推理时均使用了 `StandardScaler` 进行 $Z-Score$ 标准化，确保了不同分辨率摄像头下的兼容性。
- **串口协议**：数据以文本帧形式发送，格式为 `"#X坐标$Y坐标\r\n"`。
    - `#`：起始符。
    - `$`：分隔符。
    - `\r\n`：终止符（回车换行）。
- **波特率**：采用 `115200` 高速波特率，确保指令传输的低延迟。

## 五、 技术优势总结
1.  **端侧实时性**：SVM 的轻量级特性使得系统可以在普通 PC 甚至树莓派上流畅运行。
2.  **软硬结合**：通过串口协议打通了算法层与执行层，实现了从"像素"到"物理动作"的直接转换。
3.  **高稳定性**：多级滤波机制确保了在复杂交互场景下的硬件运行安全性。
